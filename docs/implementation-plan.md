## 전체 구조

```text
[프론트엔드 앱]                    [Supabase]
  UI · 네비 · 토스트                  DB 스키마 · RLS
  로컬 Storage (부스트·기회 등)  ←→   Edge Functions (유일한 API)
  시간 적립 "후보" 계산               원장·멱등성·금액 확정
  광고 · 토스 SDK                     기부 캠페인 · Storage
```

| 축 | 담당 문서 | 주요 범위 |
|----|-----------|-----------|
| **프론트엔드** | [`ui-layout.md`](./ui-layout.md) | 화면·UX·로컬 상태·하이브리드 계산·플랫폼 연동 |
| **Supabase** | [`supabase-data-model.md`](./supabase-data-model.md) | 스키마·Edge Function·원장·기부 로직·Storage |

**아키텍처 원칙** ([`supabase-data-model.md`](./supabase-data-model.md) 참고):

- 앱은 DB에 **직접 접근하지 않음**. **Edge Function만** 호출
- 금액·잔액의 **진실 공급원은 서버(ledger)**
- 시간 적립 등은 **클라이언트 후보 계산 → 서버 확정** 하이브리드

## 단계별 상세

### 1단계 — 브랜드·디자인 토큰

| 항목 | 내용 |
|------|------|
| **난이도** | ★☆☆☆☆ |
| **보급형 AI** | ✅ 가능 |
| **담당** | 프론트엔드 |

로고·썸네일에서 추출한 색감·라운드·분위기를 `theme` 상수로 고정합니다.

**작업 목록**

- 배경: 연한 피치/크림 (`#FDEBD0` 계열)
- Primary: 코랄/살몬 (손 색)
- Accent: 골드/노랑 (코인)
- 텍스트: 따뜻한 다크 브라운·그레이 (순수 토스 블루 탈피)
- 기존 [`src/constants/layout.ts`](../src/constants/layout.ts)의 `LAYOUT`과 함께 `theme` 모듈 구성
- 게이지 fill, CTA, 히어로 배경을 토큰으로 교체

**왜 1단계인가**

현재 코드는 토스 블루(`#3182F6`) 계열입니다. 화면을 다 만든 뒤 색을 바꾸면 전면 재작업이 됩니다. **구조 구현 전에 팔레트를 고정**하는 것이 로고·썸네일과의 통일성 유지에 핵심입니다.

| 항목 | 내용 |
|------|------|
| **난이도** | ★★☆☆☆ |
| **보급형 AI** | ✅ 가능 (레이아웃 검수 권장) |
| **담당** | 프론트엔드 |

[`ui-layout.md`](./ui-layout.md) 블록 구조를 **서버 없이** 구현합니다.

**작업 목록**

- ProgressGauge, 기부 목표 카드, 골드·기부 카드, 목록 행 등
- 숫자는 하드코딩/목(mock) 데이터
- 화면 간 네비게이션 연결

**디자인**

1단계에서 정의한 토큰만 사용하고, 새 색을 하드코딩하지 않습니다.

**레퍼런스**

| 블록 | 출처 |
|------|------|
| 히어로·부스트·4탭 상자·출석 달력 | kilomoa |
| ProgressGauge·RewardBox·시간 적립 히어로 | sansokim |
| DonationScreen | **신규** |

### 3단계 — Supabase 스키마·마이그레이션

| 항목 | 내용 |
|------|------|
| **난이도** | ★★★☆☆ |
| **보급형 AI** | ⚠️ 부분만 (초안 + 사람 검수) |
| **담당** | Supabase |

[`supabase-data-model.md`](./supabase-data-model.md) 스키마를 마이그레이션으로 구현합니다.

**작업 목록**

- `renewal_users`, `gold_ledger`, `box_ledger`, `attendance_records`, `conversions` (kilomoa 이식)

- Storage 버킷 (캠페인 이미지)

**주의**

테이블 초안은 AI가 잘 만들지만, **제약·인덱스·RLS는 사람이 검수**해야 합니다.

### 4단계 — Edge Function (kilomoa 이식 + API 계약 고정)

| 항목 | 내용 |
|------|------|
| **난이도** | ★★★★☆ |
| **보급형 AI** | ❌ 비권장 |
| **담당** | Supabase |

kilomoa `supabase/functions/renewal/`를 레퍼런스로 이식하고, gibukim 전용 엔드포인트를 추가합니다.

**kilomoa에서 가져올 것**

| 엔드포인트 | 역할 |
|------------|------|
| `POST /bootstrap` | 유저 생성·잔액·상자·활성 전환·(권장) 오늘 기부·활성 캠페인 |
| `POST /credit-boxes` | 시간 적립 상자 확정 |
| `POST /credit-box-open-gold` | 상자 열기 → 골드 |
| `POST /credit-attendance-gold` | 출석 |
| attendance month 조회 | 달력 |
| conversion 계열 | create / sdk-started / finalize / cancel / recover |

**gibukim 신규 (이 단계 또는 7~8단계에서 구현)**

| 엔드포인트 | 역할 |
|------------|------|
| `POST /donate` | 전액 기부 + 캠페인 배분·교체 루프 |
| `list-campaigns` | `is_listed=true` 목록 |
| `get-campaign-detail` | 상세 + 이미지 + 닉네임 |
| `get-my-donations` | 내 요약 (hash) |

**산출물**

요청/응답 타입을 문서로 고정 → 5단계 이후 프론트와의 **API 계약**으로 사용

## 디자인(로고·썸네일) 적용 시점

브랜드 에셋(로고·썸네일) 기준 팔레트:

| 역할 | 색감 | 용도 |
|------|------|------|
| 배경 | 연한 피치/크림 (`#FDEBD0` 계열) | 화면·카드 배경 |
| Primary | 코랄/살몬 (`#FF7F7F` 계열) | CTA, 강조, 게이지 |
| Accent | 골드/노랑 (`#FFB84D` 계열) | 코인·숫자 하이라이트 |
| 스타일 | 플랫·라운드·여백 넉넉 | 카드·버튼 border-radius |

```text
1단계  ──▶  팔레트·라운드·게이지·CTA 토큰 확정   ← 메인
2~7단계 ──▶  새 색 하드코딩 금지, 토큰만 사용     ← 유지
8단계  ──▶  빈화면·토스트·아이콘·여백 미세조정   ← 마무리
```

| 할 일 | 단계 |
|------|------|
| 피치 배경, 코랄 primary, 골드 accent 정의 | **1** |
| 카드/히어로/버튼에 토큰 적용 | **1~2** |
| 게이지·강조 숫자를 골드/코랄로 | **1**, 화면 붙일 때 |
| 로고 에셋 → 앱 아이콘·스플래시 등 | **1** 또는 스토어 준비 시 |
| 전체 톤 최종 맞춤 (빈 화면, 로딩, 토스트) | **8** |

**중간에 “디자인 전용 큰 단계”를 두지 않는 이유**

기능 화면을 토스 블루로 다 만든 뒤 브랜드 색으로 바꾸면 비용이 큽니다. **토큰을 먼저, 폴리시는 마지막**이 효율적입니다.

## 병렬 작업 축

```text
프론트:  1 → 2 ──────────→ 5 → 6 → 7 → 8
백엔드:       3 → 4 ──────↗
계약:              bootstrap / donate 스펙을 4단계 끝에서 고정
```

| 시점 | 병렬 가능 여부 |
|------|----------------|
| 1~2 (프론트) vs 3~4 (백엔드) | ✅ 동시 진행 가능 |
| 5단계 이후 | API 계약(bootstrap 등) 필요 |

## 보급형 AI 활용 가이드

### ✅ 맡겨도 됨

- 1단계: 디자인 토큰·테마 상수
- 2단계: 정적 레이아웃·목 데이터 화면
- 8단계: 기부 목록·상세 UI
- 단순 스타일 치환 (하드코딩 색 → 토큰)

### ⚠️ 초안만 + 사람 검수

- 3단계: 마이그레이션 SQL
- 5단계: API 클라이언트·에러 처리
- 8단계: 집계·노출 규칙

### ❌ 맡기지 말 것 (상위 모델 + 리뷰 필수)

- 4단계: Edge Function 원장·멱등성
- 6단계: 하이브리드 시간 적립
- 7단계: donate 트랜잭션·토스 전환
