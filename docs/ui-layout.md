## 화면 구성

| 화면 | 역할 |
|------|------|
| **HomeScreen** | 상자 적립 · 부스트 · 기부 목표 |

## 공통 디자인 그리드

kilomoa / sansokim에서 사용하던 spacing을 gibukim 기준으로 통일합니다.

| 토큰 | 값 | 용도 |
|------|-----|------|
| `screenPaddingHorizontal` | 20 | ScrollView 좌우 패딩 |
| `screenPaddingTop` | 28 | ScrollView 상단 패딩 |
| `screenPaddingBottom` | 40 | ScrollView 하단 패딩 |
| `sectionGap` | 14 | 화면 내 섹션 간 간격 |
| `cardPadding` | 18 | 일반 카드 내부 패딩 |
| `cardBorderRadius` | 24 | 일반 카드 · 광고 슬롯 |
| `heroBorderRadius` | 28 | 히어로 카드 |
| `imageBannerHeight` | 410 | 하단 InlineAd 이미지 배너 |
| `headerMarginBottom` | 18 | 앱 타이틀 아래 여백 |
| `attendanceDayCellWidthPercent` | 14.285% | 출석 달력 7열 (kilomoa 방식) |

코드 상수: `src/constants/layout.ts`

## HomeScreen

### 블록 구조

```
HomeScreen
├── [기부 목표 카드]
│   ├── [헤더] 기부처 제목만              ← 이미지 없음
│   ├── ProgressGauge (전체 모인 골드 / 목표)
│   │   └── 게이지 우측 끝: 목표 금액
│   └── 현재 모인 골드 수치
├── [히어로] 다음 상자까지 + ProgressGauge    ← sansokim
│   └── [부스트 버튼]                        ← kilomoa 히어로와 동일
├── [배너] InlineAd
├── [상자 카드]
│   ├── 골드 지갑 버튼                       ← kilomoa → PointScreen 이동
│   ├── RewardBox + x{count} + 4탭 안내      ← 공통
│   └── [상자 열기 기회 받기]                  ← sansokim
├── [배너] InlineAd
└── [공지]
```

### 기부 목표 카드 와이어프레임

```
┌─────────────────────────────────────┐
│  {기부처 제목}                        │
│                                     │
│  ████████████░░░░░░  {목표}골드      │
│  모인 골드 {현재}골드                  │
└─────────────────────────────────────┘
```

### 기부 목표 카드 규칙

| 항목 | 규칙 |
|------|------|
| 이미지 | 홈 카드에 **표시하지 않음** |
| 제목 | 헤더에 기부처 제목만 표시 |
| 목표 금액 | 게이지 **우측 끝**에 표시 |
| 모인 골드 | 모든 사용자가 기부한 골드 **총합** |
| 활성 캠페인 | **1개 고정** (`is_active = true` 1건) |
| 데이터 관리 | Supabase Table Editor에서 제목 · 목표 · 현재 골드 수정 |

### 히어로 카드 구조

sansokim 히어로 기반 + kilomoa 부스트 버튼을 게이지 아래에 배치합니다.

```
[캡션] 다음 상자까지 {시간}
[ProgressGauge]
[설명] 상자 적립 안내
[부스트 버튼]  ← kilomoa와 동일 UI/동작 (pill 형태, 활성/비활성 상태별 스타일)
```

> v1에서 계획했던 **액션 카드("오늘 골드 2배 받기")는 삭제**되었습니다. 부스트는 히어로 카드 안에 포함합니다.

## PointScreen

### 블록 구조

```
PointScreen
├── [헤더] 골드·출석
├── [배너] InlineAd
├── [골드·기부 카드]                          ← kilomoa PointConversionCard 기반
│   ├── 카드 타이틀: "골드·기부"
│   ├── [요약 2열] 보유 골드 | 오늘 기부 골드
│   ├── [배너] InlineAd
│   ├── [버튼 2열] 토스 포인트로 받기 | 기부하기
│   ├── [입력] 기부 닉네임
│   └── [안내 문구]
├── [출석] 달력 + CTA                        ← kilomoa
└── [배너] InlineAd
```

### 골드·기부 카드 와이어프레임

```
┌─────────────────────────────────────┐
│  골드·기부                           │
├─────────────────┬───────────────────┤
│  보유 골드       │  오늘 기부 골드    │
│  {N}골드         │  {N}골드           │
├─────────────────┴───────────────────┤
│  [ InlineAd ]                        │
├──────────────────┬──────────────────┤
│ 토스 포인트로 받기 │    기부하기       │
├──────────────────┴──────────────────┤
│ ┌─────────────────────────────────┐ │
│ │ 기부 닉네임                        │ │  ← placeholder, 연한 회색
│ └─────────────────────────────────┘ │
├─────────────────────────────────────┤
│ · 기부 닉네임을 입력한 뒤 기부하기를   │
│   눌러주세요. (최대 10자)             │
│ · 목표 골드가 달성되면 실제 기부로     │
│   이어집니다.                        │
│ · 기부는 하루기부의 두리여유법인을     │
│   통해 진행되는 법인 기부입니다.       │
└─────────────────────────────────────┘
```

### kilomoa 토스 포인트 카드 대비 변경점

| kilomoa (현재) | gibukim (변경) |
|----------------|----------------|
| 카드 타이틀: "토스 포인트" | **"골드·기부"** |
| 우: 오늘 받은 포인트 | **오늘 기부 골드** (당일 기부한 골드만) |
| CTA 1개: 토스 포인트 받기 | **2개 좌/우**: 토스 포인트로 받기 / 기부하기 |
| policyNoticeBox | 기부 목표 달성 · 법인 기부 안내 + 닉네임 입력 안내 |
| — | **기부 닉네임 입력칸** 추가 (버튼 아래) |

### 입력 · 버튼 규칙

| 항목 | 규칙 |
|------|------|
| 카드 타이틀 | **"골드·기부"** |
| 오늘 기부 골드 | **당일 기부한 골드만** 집계 · 표시 |
| 기부 닉네임 placeholder | `"기부 닉네임"` (연한 회색, `#B0B8C1` 계열) |
| 닉네임 최대 길이 | **10자 (띄어쓰기 포함)** |
| 닉네임 중복 | **허용** |
| 기부하기 활성 조건 | 닉네임 trim 후 1자 이상 |
| 토스 포인트로 받기 | kilomoa `REQUEST_POINT_CONVERSION`과 동일 흐름 |

### 삭제된 블록

- MetricGauge 카드 (보유 골드 / 보유 상자) — 골드·기부 카드로 통합
- 보너스 카드 ("광고 보고 1골드 받기")
- kilomoa 토스 포인트 전환 카드 단독 구조
- kilomoa 응원 피드 카드

## 기부하기 UX

**모달 · 축하 팝업 없음. 성공 · 실패 모두 토스트만 사용합니다.**

### 성공 플로우

```
[기부하기 탭]
  → 버튼 로딩 상태 (비활성 + 로딩 표시)
  → 서버 기부 API 성공
  → 보유 골드 / 오늘 기부 골드 즉시 갱신
  → 성공 토스트 표시
  → 닉네임 입력칸 초기화
  → 기부하기 버튼 다시 비활성 (닉네임 없음 상태)
```

### 성공 토스트 문구 (확정)

```
🌱 "{닉네임}님의 {N}골드로 오늘 하루가 조금 더 따뜻해졌습니다."
```

| 변수 | 설명 |
|------|------|
| `{닉네임}` | 입력한 기부 닉네임 |
| `{N}` | 이번에 기부한 골드 수량 |

### 중복 탭 방지

| 상태 | 기부하기 버튼 |
|------|---------------|
| 닉네임 없음 | 비활성 |
| API 요청 중 | **비활성 + 로딩** |
| 성공 후 | 닉네임 초기화 → 비활성 |
| 실패 후 | 다시 활성 (닉네임 유지) |

### 실패 시 토스트

| 실패 원인 | 토스트 문구 |
|-----------|------------|
| 골드 부족 | `보유 골드가 부족해요. 골드를 모은 뒤 다시 시도해 주세요.` |
| 네트워크 오류 | `네트워크 연결을 확인하고 다시 시도해 주세요.` |
| 서버 오류 | `잠시 후 다시 시도해 주세요.` |
| 닉네임 유효성 (10자 초과 등) | `닉네임은 10자 이내로 입력해 주세요.` |
| 기타 | `기부에 실패했어요. 다시 시도해 주세요.` |

### UX 원칙

| 원칙 | 내용 |
|------|------|
| 피드백 방식 | **토스트만** (모달 · 축하 팝업 없음) |
| 신뢰감 | **오늘 기부 골드 · 보유 골드 즉시 갱신**이 주 피드백 |
| 안전 | 로딩 중 **중복 탭 방지** |
| 실패 | 원인별 **토스트**로 안내 |

## DonationScreen

### 블록 구조

```
DonationScreen
├── [내 기부 요약]
│   ├── 참여한 기부 개수
│   └── 총 기부 골드량
├── [기부 목록]
│   └── 항목: [썸네일] + [제목] 만
└── (탭) → DonationDetail

DonationDetail
├── [기부 이미지] (대표 이미지)
├── [기부 정보]
│   ├── 제목
│   └── 총 기부금액
└── [참여자 목록]
    └── 기부 시 입력한 닉네임
```

### 목록 항목 와이어프레임

```
┌──────┬──────────────────────────┐
│ 썸네일 │  {기부 제목}               │
│ 56px │                          │
└──────┴──────────────────────────┘
```

| 표시 | 미표시 |
|------|--------|
| 썸네일 (56~64px, 좌측 정사각) | 기간 |
| 제목 | 달성률 |
| | 금액 |

> 목록 썸네일은 캠페인 **커버 이미지**를 사용합니다. 상세는 다중 이미지. 홈 기부 목표 카드에서는 이미지를 표시하지 않습니다. 스키마는 [`supabase-data-model.md`](./supabase-data-model.md).

### 상세 화면

| 블록 | 내용 |
|------|------|
| 기부 이미지 | 캠페인 다중 이미지 (커버 포함) |
| 기부 정보 | **제목**, **총 기부금액** |
| 참여자 목록 | 기부 시 입력한 **닉네임** (중복 허용) |

## Supabase 데이터 모델

서버 원장 · 해시 · 기부 캠페인 · donate 트랜잭션 · 시간 상자 적립의 **최종 스펙**은  
[`supabase-data-model.md`](./supabase-data-model.md)를 따릅니다.

화면에서 쓰는 요약만 적습니다.

| 화면 | 데이터 |
|------|--------|
| 홈 기부 목표 카드 | 활성 캠페인 `title`, `goal_gold`, `current_gold` (이미지 없음) |
| 기부 목록 | `is_listed=true` · **커버 이미지**(썸네일) + `title` |
| 기부 상세 | 다중 이미지 · `title` · `current_gold` · 참여자 `nickname` |
| 내 기부 요약 | `donation_participants`를 `anonymous_hash`로 집계 |
| 오늘 기부 골드 | 당일 KST + hash 합 |
| 기부 금액 | 서버 **전액 기부** (클라 금액 입력 없음) |

운영·스키마·목표 달성 자동 교체·플레이스홀더「다음 기부」는 `supabase-data-model.md` 참고.
